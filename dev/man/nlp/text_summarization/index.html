Text Summarization

This section will demonstrate how to use TextAnalysis.jl (Julia's leading NLP library) for QuranTree.jl. In particular, in summarizing the Qur'an, specifically Chapter 18 (The Cave) which most Muslims are familiar with (this is the chapter recommended to be read every Friday). The algorithm used for summarization is called TextRank, an application of PageRank algorithm to text datasets.

julia> using QuranTree
julia> using TextAnalysis
julia> using Yunir
julia> crps, tnzl = QuranData() |> load;

Data Preprocessing

The first data processing will be the removal of all Disconnected Letters (like الٓمٓ ,الٓمٓصٓ, among others), Prepositions, Particles, Conjunctions, Pronouns, and Adverbs. This is done as follows:

julia> function preprocess(s::String)
           feat = parse(QuranFeatures, s)
           disletters = isfeat(feat, AbstractDisLetters)
           prepositions = isfeat(feat, AbstractPreposition)
           particles = isfeat(feat, AbstractParticle)
           conjunctions = isfeat(feat, AbstractConjunction)
           pronouns = isfeat(feat, AbstractPronoun)
           adverbs = isfeat(feat, AbstractAdverb)
       
           return !disletters && !prepositions && !particles && !conjunctions && !pronouns && !adverbs
       endpreprocess (generic function with 1 method)
julia> crpstbl = filter(t -> preprocess(t.features), crpsdata[18].data)ERROR: DomainError with QuranFeatures("STEM|POS:CERT|LEM:qad"):
Expected SUFFIX feature, got STEM.

Next, we create a copy of the above data (so we have the original state), and use the copy to do further data processing.

julia> crpsnew = deepcopy(crpstbl)ERROR: UndefVarError: crpstbl not defined
julia> feats = crpsnew[!, :features]ERROR: UndefVarError: crpsnew not defined

Lemmatization

Using the above parsed features, we then convert the form of the tokens into its lemma. This is useful for addressing variations due to inflection.

julia> lemmas = lemma.(feats)ERROR: UndefVarError: feats not defined
julia> forms1 = crpsnew[!, :form]ERROR: UndefVarError: crpsnew not defined
julia> forms1[.!ismissing.(lemmas)] = lemmas[.!ismissing.(lemmas)]ERROR: UndefVarError: lemmas not defined
Tips

We can also use the Root features instead, which is done by simply replacing lemma.(feats) with root.(feats).

We now put back the new form to the corpus:

julia> crpsnew[!, :form] = forms1ERROR: UndefVarError: crpsnew not defined
julia> crpsnew = CorpusData(crpsnew)ERROR: UndefVarError: crpsnew not defined

Tokenization

We want to summarize the Qur'an at the verse level. Thus, the token would be the verses of the corpus. From these verses, we further clean it by dediacritization and normalization of the characters:

julia> lem_vrs = verses(crpsnew)ERROR: UndefVarError: crpsnew not defined
julia> vrs = normalize.(dediac.(lem_vrs))ERROR: UndefVarError: lem_vrs not defined

Creating a TextAnalysis Corpus

To make use of the TextAnalysis.jl's APIs, we need to encode the processed Quranic Corpus to TextAnalysis.jl's Corpus. In this case, we will create a StringDocument of the verses.

julia> crps1 = Corpus(StringDocument.(vrs))ERROR: UndefVarError: vrs not defined

We then update the lexicon and inverse index for efficient indexing of the corpus.

julia> update_lexicon!(crps1)ERROR: UndefVarError: crps1 not defined
julia> update_inverse_index!(crps1)ERROR: UndefVarError: crps1 not defined

Next, we create a Document Term Matrix, which will have rows of verses and columns of words describing the verses.

julia> m1 = DocumentTermMatrix(crps1)ERROR: UndefVarError: crps1 not defined

TF-IDF

Finally, we compute the corresponding TF-IDF, which will serve as the feature matrix.

julia> tfidf = tf_idf(m1)ERROR: UndefVarError: m1 not defined

Summarizing the Qur'an

Using the TF-IDF, we compute the product of it with its transpose to come up with a square matrix, where the elements describes the linkage between the verses, or the similarity between the verses.

julia> sim_mat = tfidf * tfidf'ERROR: UndefVarError: tfidf not defined

At this point, we can now write the code for the PageRank algorithm:

julia> using LinearAlgebra
julia> function pagerank(A; Niter=20, damping=.15)
           Nmax = size(A, 1)
           r = rand(1, Nmax);              # Generate a random starting rank.
           r = r ./ norm(r, 1);            # Normalize
           a = (1 - damping) ./ Nmax;      # Create damping vector
       
           for i=1:Niter
               s = r * A
               rmul!(s, damping)
               r = s .+ (a * sum(r, dims=2));   # Compute PageRank.
           end
       
           r = r ./ norm(r, 1);
       
           return r
       endpagerank (generic function with 1 method)

Using this function, we apply it to the above similarity matrix (sim_mat) and extract the PageRank scores for all verses. These scores will serve as the weights, and so higher scores suggest that the verse has a lot of connections to other verses in the corpus, which means it represents per se the corpus.

julia> p = pagerank(sim_mat)ERROR: UndefVarError: sim_mat not defined

Now we sort these scores in descending order and use it to rearrange the original verses:

julia> idx = sortperm(vec(p), rev=true)[1:10]ERROR: UndefVarError: p not defined

Finally, the following 10 verses best summarizes the corpus (Chapter 18) using TextRank:

julia> verse_nos = verses(CorpusData(crpstbl), number=true, start_end=false)ERROR: UndefVarError: crpstbl not defined
julia> verse_out = String[];
julia> chapter = Int64[];
julia> verse = Int64[];
